
# обертка над vpn для биллинга
С покупкой ключей через telegram-бота  
С хранением данных в формате "id-юзера:время-окончания" в ДБ  
# АРХИТЕКТУРА
Есть сервер с БД. на нем крутится тг-бот, через которого пользователи получают ключи. когда добавляется новый юзер, то этот первый сервер отправляет данные на сервер с vpn, там все настраивается, и отвечает vpn-ключом, который затем шлется тг-ботом
